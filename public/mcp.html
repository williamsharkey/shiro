Shiro Remote MCP

Connect Claude Code to your browser-based Shiro instance via WebRTC

What is this?

Shiro Remote lets you control a Shiro browser instance directly from Claude Code running on your computer. No servers needed — it uses WebRTC for a direct peer-to-peer connection.

Quick Start

1 In Shiro (browser) — Start a remote session:

remote start

This generates a connection code like fluffy-cloud-shimutako and copies it to your clipboard.

2 On your computer — Add the MCP server to Claude Code:

# Add to ~/.claude/claude_desktop_config.json
{
  "mcpServers": {
    "shiro": {
      "command": "npx",
      "args": ["-y", "shiro-mcp"]
    }
  }
}

3 In Claude Code — Connect using the code:

Use the connect_shiro tool with code "fluffy-cloud-shimutako"

4 Done! Now you can use Shiro tools:

Use shiro_exec to run "ls -la"
Use shiro_read to read "/home/user/.bashrc"

Available Tools

All tools are namespaced under shiro: in Claude Code.

connect
Connect to a Shiro instance using a connection code
disconnect
Disconnect from the current Shiro instance
status
Check the current connection status
exec
Execute a shell command (ls, cat, git, npm, etc.)
read
Read a file from the virtual filesystem
write
Write a file to the virtual filesystem
list
List directory contents
eval
Execute JavaScript in the browser context

Connection Codes

Connection codes like fluffy-cloud-shimutako provide ~46 bits of entropy, making them virtually impossible to guess. They expire after 5 minutes.

The code has three parts:

  • Adjective (200 options) — fluffy, cosmic, brave...
  • Noun (200 options) — cloud, fox, crystal...
  • Syllables (64⁴ options) — shimutako, karinobe...

Only the first two parts are shown in the Shiro banner for privacy — someone watching your screen can't hijack your session.

Manual Installation

If you prefer to install manually:

# Install globally
npm install -g shiro-mcp

# Or clone and run from source
git clone https://github.com/williamsharkey/shirocode
cd shirocode
npm install
npm run mcp:remote
Note

The MCP server requires the wrtc package for WebRTC support in Node.js. This is included as a dependency and should install automatically.

Shiro Commands

Commands available in the Shiro terminal:

remote start    # Start a remote session, get connection code
remote stop     # Stop the current remote session
remote status   # Check connection status
clear           # Refresh the banner to show/hide connection code

Troubleshooting

Connection fails immediately

Make sure the code hasn't expired (5 minute timeout). Run remote start again in Shiro to get a fresh code.

WebRTC connection times out

Some corporate networks block WebRTC. Try from a different network or use a VPN.

"wrtc not installed" error

The wrtc package requires native compilation. On some systems you may need build tools:

# macOS
xcode-select --install

# Ubuntu/Debian
sudo apt install build-essential python3

# Then reinstall
npm install wrtc
